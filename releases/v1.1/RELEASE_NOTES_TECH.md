# v1.1 升级说明（技术版）

## 目标
相对 v1.0，v1.1 的核心目标是：
- 建立完整认证与权限体系。
- 提升后台运营能力（筛选、统计、分享、配置热更新）。
- 提升前后端可维护性与移动端可用性。

## 关键变更

## 1) 认证与权限体系
- 新增用户注册、登录、登出、当前用户查询、修改密码。
- 引入基于会话的访问控制，中间件区分登录与管理员权限。
- 管理端接口统一受保护。

## 2) 管理后台能力增强
- Admin 列表新增筛选参数：status/category/keyword/include_expired。
- Admin 返回聚合 summary（total/visible/byStatus/byCategory 等）。
- 新增用户管理相关接口与页面。

## 3) 表单治理模型（配置层）
- Form 配置扩展字段：
  - category
  - pinned
  - sort_order
  - priority
  - status
  - publish_at
  - expire_at
- 前台与后台统一使用治理字段进行展示与排序。

## 4) 专用分享链路
- 新增管理员生成分享链接接口：
  - POST /api/admin/share-links
- 新增公开读取与提交接口：
  - GET /api/public/forms/{token}
  - POST /api/public/submit/{token}
- 支持无需登录直达单表单填写。

## 5) 在线 YAML 编辑与热重载
- 新增配置读取与保存接口：
  - GET /api/admin/form-config/{formName}
  - PUT /api/admin/form-config/{formName}
- 保存后触发表单配置热重载。
- 增加并发保护与配置路径安全校验，避免热更新期数据竞争与越界写入。

## 6) 配置体系升级（多文件合并）
- 主配置支持 includes（含通配符）加载多个 YAML。
- 记录配置来源（ConfigSource），方便排查与后台编辑。

## 7) 数据层与表结构管理
- 持续收敛到统一数据库存储。
- 启动阶段按配置进行建表/补列，降低人工维护成本。

## 8) 前端架构与交互
- Vue 路由完善：登录、注册、改密、我的提交、管理页、分享页。
- 首页从卡片逐步优化到列表/表格化展现。
- 管理端筛选区、表格列对齐、弹窗交互与移动端体验持续优化。

## 运维与部署变化
- 建议监听地址使用 0.0.0.0:8080，便于局域网访问。
- 分享链接生成支持动态使用服务器 IPv4（避免固定 localhost）。

## 升级检查清单
1. 校验 config.yaml 与 includes 引用是否完整。
2. 校验所有表单 name 与 table_name 唯一性。
3. 校验管理员账号与密码策略。
4. 校验分享链接在局域网访问可达。
5. 校验移动端关键页面：首页、管理筛选、表单填写。

## 回滚建议
- 保留 v1.0 二进制与配置快照。
- 升级前备份 data/data.db。
- 若异常，回滚二进制 + 配置 + 数据库备份可快速恢复。
